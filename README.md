# 面试管理系统

## 项目简介

面试管理系统是一个集成了面试日程安排、企业邮件状态跟踪和投递汇总功能的一站式求职管理工具。该系统可以帮助求职者更好地管理秋招过程中的各项事务。

## 功能特性

### 1. 面试日程安排
- 管理面试日程，包括公司、岗位、时间等信息
- 标记面试准备状态和完成情况
- 支持按时间排序面试记录
- 自动保存数据到本地数据库
- 响应式设计，支持移动端访问

### 2. 企业邮件跟踪
- 自动获取并分类企业发送的各类邮件（面试邀请、笔试通知、评测结果等）
- 支持按关键词搜索邮件
- 支持按日期范围筛选邮件
- 支持查看邮件正文内容
- 支持删除不需要的邮件记录
- 支持批量删除邮件
- 与投递汇总系统联动，一键添加到投递记录

### 3. 企业投递汇总
- 记录和管理企业投递情况
- 跟踪投递状态（已投递、简历筛选中、一面通过等）
- 支持添加、编辑和删除投递记录
- 支持按时间排序投递记录
- 响应式设计，支持移动端访问

### 4. 系统配置管理
- 图形化配置界面，方便录入和修改系统配置
- 配置信息持久化存储在浏览器localStorage中
- 支持数据库和邮箱配置的动态修改
- 无配置时自动跳转到配置页面

## 技术架构

### 前端技术栈
- HTML5 + CSS3 + JavaScript (原生实现，无框架依赖)
- 响应式设计，支持移动端访问
- 使用localStorage存储配置信息
- 统一的UI组件样式设计

### 后端技术栈
- Node.js + Express (轻量级Web服务器)
- MySQL数据库存储业务数据
- RESTful API设计风格

### 邮件处理模块
- Python脚本处理邮件获取和解析
- IMAP协议连接邮箱服务器
- 支持QQ邮箱等主流邮件服务商

## 安装与部署

### 环境要求
- Node.js (v12+)
- Python (v3.6+)
- MySQL (v5.7+)

### 安装步骤

1. **克隆项目代码**
   ```bash
   git clone <项目地址>
   cd 面试管理系统
   ```

2. **安装Node.js依赖**
   ```bash
   npm install
   ```

3. **确保MySQL服务已启动**
   - 启动MySQL服务
   - 确保可以通过localhost访问MySQL

4. **启动服务器**
   ```bash
   npm start
   ```

5. **访问系统**
   - 打开浏览器访问 http://localhost:3001
   - 首次使用时会自动跳转到配置页面

### 系统配置

首次访问系统时需要进行基础配置：

1. 数据库配置：
   - DB_HOST: MySQL服务器地址（本地为localhost）
   - DB_USER: MySQL用户名
   - DB_PASSWORD: MySQL密码
   - DB_CHARSET: 数据库字符集（推荐utf8mb4）

2. 邮箱配置：
   - EMAIL_ADDRESS: 用于接收企业邮件的邮箱地址
   - EMAIL_PASSWORD: 邮箱授权码（QQ邮箱需要开启IMAP服务并获取授权码）
   - IMAP_SERVER: 邮箱IMAP服务器地址（QQ邮箱为imap.qq.com）

## 使用说明

### 首次配置系统
1. 首次访问系统时会自动跳转到配置页面
2. 输入数据库连接信息和邮箱配置信息
3. 点击"保存配置"按钮保存配置
4. 配置信息将保存在浏览器的localStorage中

### 面试日程管理
1. 访问面试日程页面 (http://localhost:3001/schedule.html)
2. 添加新的面试信息或编辑现有信息
3. 标记准备状态和完成情况
4. 支持按时间排序查看面试记录

### 邮件状态跟踪
1. 访问企业邮件跟踪页面 (http://localhost:3001/emails.html)
2. 设置日期范围并点击"获取邮件"按钮获取最新邮件
3. 使用搜索功能查找特定邮件
4. 点击"查看正文"查看邮件详细内容
5. 支持与投递汇总系统联动，一键添加到投递记录

### 投递汇总管理
1. 访问企业投递汇总页面 (http://localhost:3001/deliveries.html)
2. 添加新的投递记录或编辑现有记录
3. 跟踪投递状态变化
4. 支持按时间排序查看投递记录

### 修改系统配置
1. 访问系统配置页面 (http://localhost:3001/config.html)
2. 修改相应的配置信息
3. 点击"保存配置"按钮保存修改

## 配置说明

系统使用浏览器localStorage存储配置信息，避免敏感信息硬编码。首次使用时需要配置必要的系统参数：

### 数据库配置说明
- DB_HOST: MySQL服务器地址
- DB_USER: MySQL用户名
- DB_PASSWORD: MySQL密码
- DB_CHARSET: 数据库字符集

### 邮箱配置说明
- EMAIL_ADDRESS: 用于接收企业邮件的邮箱地址
- EMAIL_PASSWORD: 邮箱授权码（QQ邮箱需要开启IMAP服务并获取授权码）
- IMAP_SERVER: 邮箱IMAP服务器地址（QQ邮箱为imap.qq.com）

## 数据库结构

系统使用三个数据库：
1. `interview_schedule` - 存储面试日程信息
2. `job_emails` - 存储企业邮件信息，按分类建立不同表
3. `job_deliveries` - 存储投递汇总信息

## 注意事项

1. 邮件获取功能需要确保邮箱开启了IMAP服务并获取了授权码
2. 系统会自动创建所需的数据库和表结构
3. 数据会自动保存到本地数据库，无需手动备份
4. 首次使用时请务必配置数据库和邮箱信息
5. QQ邮箱需要使用授权码而非邮箱密码进行登录
6. 配置信息存储在浏览器localStorage中，清除浏览器数据会丢失配置

1. 前端页面已优化为使用公共CSS文件，减少代码重复

4. 敏感配置信息通过localStorage管理，便于开源部署
5. 提供图形化配置界面，简化系统部署和维护
